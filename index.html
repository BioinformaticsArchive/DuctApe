Analyze and link genomics and phenomics experiments

DuctApe is a python library with a series of command-line programs that will help bioinformaticians to analyze genomes AND phenomic experiments.
The final purpose of the program is to combine the genomic informations (encoded as KEGG pathways) with the results of phenomic experiments (for now the Phenotype microarrays are supported) and highlight the genes that may be responsible for phenotypic variations.

Features

Powerful
Many kind of experiment are supported: single organism, mutants, pangenome
Flexible
Three command-line tools with git-like syntax, to allow maximum flexibility in the analysis
Modular
Each component of DuctApe is encapsulated in a thread, to simplify further developments of client applications (i.e. GUIs or web interfaces)
Portable
All the analysis are saved in a single file (a SQLite database), to avoid complexity and to encourage exchange of data

The three modules of DuctApe

  • dape

    • Used to initialize the project and add/remove the organism
    • Performs the combination of genomics and phenomics experiments (*)

  • dgenome

    • Handles protein sequences for each organism of the project
    • For pangenomic studies it can build the pangenome using the BBH algorithm (user-defined pangenomes can be added)
    • The proteins are mapped to KEGG metabolic pathways

  • dphenome

    • Handles phenomic experiments for each organism of the project (Phenotype microarray experiments)
    • Performs zero-subtraction on the biolog plates
    • Calculates the growth parameters
    • Ranks the single experiments with a clusterization over growth parameters
    • The compounds from the phenomic experiments are mapped to KEGG


(*) not implemented yet

Quick how-to


Requirements

DuctApe has a few python library dependencies:
  • Biopython
  • NumPy
  • SciPy
  • matplotlib
  • scikits.learn
  • SOAPpy
  • multiprocessing
  • networkx (*)

Installation

  • Method 1 (dependencies won't be checked)
    • sudo python setup.py install
  • Method 2 (missing dependencies will be downloaded)
    • sudo pip install DuctApe-X.X.X.tar.gz

Usage examples

  1. Single organism experiment

    2. # Initialize the project
dape init
# add my organism using the ID MyOrg
dape add MyOrg

# add the proteome of MyOrg
dgenome add MyOrg.faa MyOrg
# adds the output of KAAS, a KEGG mapper
dgenome add-ko MyOrg.tab
# map the proteome to KEGG
dgenome start
# output the KEGG metabolic maps
dgenome map
# statistics and graphics 
dgenome stats
# export the genomic data
dgenome export

# add the phenomic experiment, BIOLOG data
dphenome add MyOrg.csv MyOrg
# perform control subtraction
dphenome zero
# calculate the growth parameters and perform the clusterization, using 4 CPUs
dphenome start -n 4
# plot the growth curves
dphenome plot
# remove inconsistent replicas: keep the highest replicas when there is an activity index delta >= 3
dphenome purge -d 3 keep-max
# plot only those curves that are not purged
dphenome plot
# restore the purged replicas
dphenome restore
dphenome stats
dphenome export

dape map (*)

  2. Mutant experiment

    3. dape init
dape add MyOrg
# add mutant MyMut, a deletion mutant of MyOrg
dape add-mut -m MyOrg -k deletion MyMut

# add the proteins files found in this directory
dgenome add-dir MyFolder
dgenome add-ko MyOrg.tab
dgenome start
# plot only tha maps of the mutant
dgenome map MyMut
# add the phenomic files found in this directory
dphenome add-dir MyPhenomicFolder
dphenome zero
dphenome start -n 4
dphenome purge -d 3 keep-max
dphenome plot

dape map (*)

  3. Pangenomic experiment

    4. dape init
dape add MyOrg
dape add MyOrg2
dape add MyOrg3

dgenome add-dir MyFolder
dgenome add-ko MyOrg.tab MyOrg2.tab MyOrg3.tab
# also perform pangenome creation using 4 CPUs
dgenome start -n 4
# plot the maps for the whole pangenome
dgenome map

dphenome add-dir MyPhenomicFolder
dphenome zero
dphenome start -n 4
dphenome purge -d 3 keep-max
dphenome plot

dape map (*)

(*) not implemented yet
